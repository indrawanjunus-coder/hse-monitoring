Passwords hashed with SHA-256 + fixed global pepper
File
artifacts/api-server/src/lib/auth.ts:76
Evidence
createHash("sha256").update(password + "hse_salt_2024").digest("hex")
Risk
SHA-256 is a fast hash (billions/sec on GPU). With a global salt (same for every user), a single rainbow table cracks every password in the DB after any leak.
Fix
Switch to argon2id (via argon2 npm package) or bcrypt with cost ≥12. Migrate existing hashes on next successful login (verify old → rehash with new).

H4Cross-tenant data leak — "if companyId null, fetch everything" pattern [A01:2025 · API1:2023]
Files
routes/dashboard.ts:18-19,124-125, routes/users.ts:27-28, routes/schedules.ts:101-102, routes/inspections.ts:40-42
Evidence
const all = cid
  ? await db.select().from(X).where(eq(X.companyId, cid))
  : await db.select().from(X);
Risk
The "no cid" branch is intended for sysadmin (companyId: null). Combined with H1, any attacker can craft a token with companyId: null and pull every incident / user / schedule / inspection across every tenant. Even without H1, a legitimate sysadmin accessing a company URL gets mixed-tenant data.
Fix
Replace the cid == null check with an explicit role === "sysadmin" check. Reject requests on company-scoped endpoints when the caller's role is not sysadmin and cid is null.

H6Server-side HTML injection into outbound email [A05:2025 Injection]
File
lib/email.ts:205-213 (template incidentTargetReminderHtml), and sibling templates
Evidence
<p>${incident.detail}</p>
... ${reporterName} ... ${categoryName} ... ${plantName}
// all user-sourced, no escape / sanitize
Risk
An employee reports an incident with detail: "<a href='https://phish.example'>Click here</a>" — the reminder email to the PIC group (often executives) renders the HTML. Phishing, UI spoofing, tracking pixels. Scripts are blocked by most mail clients but links, images, and styles are not.
Fix
HTML-escape every ${value} in templates, or run through sanitize-html / DOMPurify (server build). Fast fix: a 6-line escaper util.
